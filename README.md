# 📘 База знаний — онлайн-библиотека с защищённой авторизацией

> Гайд от Тёмы. Делаем защищённую онлайн-библиотеку, где есть один главный босс (Owner), пара админов и обычные юзеры. С шифрованием паролей, блокировкой брутфорса и серверной защитой данных.

---

## 🎯 Что ты получишь

- 🌐 Онлайн-библиотеку документов (PDF, Word, PowerPoint, ссылки)
- 🔐 Реальную серверную защиту через **Firebase**
- 👑 Одного главного — **Owner** (его нельзя удалить/заблокировать никому)
- 👥 Роли: **Owner / Admin / User**
- 🔍 Умный поиск с релевантностью
- ⏱️ Блокировка после 5 неудачных попыток входа (на 15 мин)
- 🚪 Авто-выход через 30 минут бездействия
- 🎨 Три темы оформления (светлая / тёмная / деловая)
- 📱 Работает на телефоне, планшете, десктопе

## 🚀 Быстрый старт (TL;DR)

Для тех, кто уже шарит:

1. Создай Firebase-проект + Email/Password auth + Firestore
2. Залей `firestore.rules` в раздел Rules
3. Скопируй конфиг в `js/firebase-config.js`
4. Запусти `python -m http.server 8080` в папке проекта
5. Открой `http://localhost:8080` → создай Owner

## 📝 Пошаговая инструкция

### Шаг 1. Создай Firebase-проект

1. Открой https://console.firebase.google.com/
2. Нажми большую кнопку **«Добавить проект»** (или «Add project»)
3. Придумай имя (например, `my-library-2026`) → **Продолжить**
4. Google Analytics — **выключи** (он не нужен) → **Создать проект**
5. Подожди ~30 сек → **Продолжить**

> 💡 Имя проекта будет в URL. Лучше латиницей, без пробелов.

### Шаг 2. Включи вход по email и паролю

1. В меню слева найди **Build** (или «Сборка») → **Authentication**
2. Нажми **«Начать»** (Get started)
3. Откроется вкладка **«Sign-in method»** (Метод входа)
4. В списке провайдеров кликни **«Email/Password»**
5. Переключатель **Enable** → ON → **Сохранить**

### Шаг 3. Создай базу данных Firestore

1. В меню слева: **Build → Firestore Database**
2. Нажми **«Создать базу данных»** (Create database)
3. Выбери **«Начать в тестовом режиме»** (Start in test mode) → **Далее**
4. Выбери регион — рекомендую `eur3 (europe-west)` (ближе к России) → **Включить**

### Шаг 4. Залей правила безопасности

1. В Firestore открой вкладку **«Rules»** (Правила)
2. **Удали** всё что там есть
3. Открой файл `firestore.rules` из проекта в блокноте
4. **Скопируй** всё содержимое → **вставь** в редактор правил
5. Нажми **«Опубликовать»** (Publish)

> 🛡️ Это самая важная часть. Эти правила не дают обычным юзерам менять роли, удалять Owner, и т.д.

### Шаг 5. Получи конфиг Firebase

1. В Firebase Console нажми на **шестерёнку ⚙️** → **«Настройки проекта»** (Project settings)
2. Прокрути вниз до секции **«Твои приложения»** (Your apps)
3. Нажми на иконку **`</>`** (это «добавить веб-приложение»)
4. Введи nickname (например, `library`) → **«Зарегистрировать приложение»**
5. **Скопируй** весь объект `firebaseConfig` — он выглядит примерно так:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyDxxxxxxxxxxxxxxxxxxxxxxxxx",
  authDomain: "my-library-2026.firebaseapp.com",
  projectId: "my-library-2026",
  storageBucket: "my-library-2026.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abcdef1234567890"
};
```

### Шаг 6. Заполни `firebase-config.js`

1. Открой файл `js/firebase-config.js` в блокноте (или VS Code)
2. Замени **всё** содержимое `window.firebaseConfig = { ... }` на свой конфиг
3. Сохрани файл

Должно получиться что-то вроде:

```javascript
window.firebaseConfig = {
  apiKey: "AIzaSyDxxxxxxxxxxxxxxxxxxxxxxxxx",       // ← сюда вставил
  authDomain: "my-library-2026.firebaseapp.com",   // ← сюда вставил
  projectId: "my-library-2026",                   // ← сюда вставил
  storageBucket: "my-library-2026.appspot.com",   // ← сюда вставил
  messagingSenderId: "123456789012",              // ← сюда вставил
  appId: "1:123456789012:web:abcdef1234567890"    // ← сюда вставил
};
```

### Шаг 7. Запусти локальный сервер

> ⚠️ Firebase **не работает** если просто открыть `index.html` двойным кликом. Нужен локальный сервер.

**Вариант A: Python** (если установлен)
```bash
cd D:\site
python -m http.server 8080
```

**Вариант B: Node.js** (если установлен)
```bash
cd D:\site
npx serve -p 8080
```

**Вариант C: VS Code + Live Server**
1. Установи расширение «Live Server» (автор: Ritwick Dey)
2. Открой папку `D:\site` в VS Code
3. Правый клик по `index.html` → «Open with Live Server»

> 💡 В терминале должно появиться что-то вроде `Server started at http://localhost:8080`. **Не закрывай** это окно, пока работаешь с сайтом.

### Шаг 8. Открой сайт

В браузере зайди на **http://localhost:8080**

Должен открыться экран **«Создать владельца системы»**.

### Шаг 9. Создай Owner-аккаунт

1. **Email** — используй свой реальный email (на него придут уведомления)
2. **Пароль** — минимум 12 символов, разные регистры, цифры, спецсимволы
   - ✅ Хорошо: `My$ecretL1b22!`, `Pa$$w0rd_2026_AAA`
   - ❌ Плохо: `123456`, `password`, `admin123`
3. Повтори пароль
4. Нажми **«Создать владельца»**

Готово! Ты залогинен как 👑 **Owner**.

### Шаг 10. Добавь сотрудников

1. Перейди на вкладку **«Пользователи»** (видна только Owner/Admin)
2. Заполни:
   - Email сотрудника
   - Пароль (минимум 12 символов)
   - Роль: Пользователь или Админ
3. Нажми **«Добавить»**

Сотрудник сможет войти со своим email/паролем.

## 🛡️ Безопасность — что защищено

| Угроза | Защита |
|--------|--------|
| Пароль в открытом виде | Firebase хэширует bcrypt'ом (нельзя прочитать) |
| Подбор пароля (брутфорс) | Блокировка после 5 попыток на 15 мин |
| Угон сессии | Firebase JWT с авто-истечением |
| Подмена роли через консоль | Firestore Rules запрещают менять `role` |
| Удаление Owner | Rules запрещают удалять `role: "owner"` |
| Чтение БД неавторизованным | Rules запрещают без `request.auth` |
| XSS-атаки | Все данные экранируются через `escapeHtml()` |
| Чужой код на странице | Firebase защищается правилами, а не сокрытием ключа |

> 🔒 Пароли в коде **не лежат** — Firebase сам хэширует на сервере. Ключи в `firebase-config.js` безопасны для публикации (это нормальная практика).

## 🆘 Проблемы и решения

### «Неверный логин/пароль» сразу после создания Owner

Проверь:
- В `js/firebase-config.js` правильный `projectId`?
- В Firebase Console → Authentication → есть юзер с твоим email?

### «Permission denied» при добавлении документа

Проверь:
- Залил ли ты `firestore.rules` в Firebase Console → Firestore → Rules?
- Опубликовал ли правила (кнопка «Publish»)?

### «Firebase: Error (auth/invalid-api-key)»

В `firebase-config.js` неправильный `apiKey`. Перепроверь.

### Открываю `index.html` напрямую — не работает

Нужен локальный сервер. Читай **Шаг 7**.

### Забыл пароль Owner

1. Firebase Console → Authentication → выбери юзера
2. Три точки ⋮ → **«Reset password»**
3. На email придёт письмо со ссылкой для сброса

### Случайно удалил Owner из Firebase Console

Это катастрофа, но можно восстановить:
1. Зарегистрируй нового юзера в Authentication (любой email/пароль)
2. В Firestore → `config/owner` → отредактируй `ownerUid` на нового
3. В Firestore → `users/{новый_uid}` → создай документ с `role: "owner"`

## 🎨 Кастомизация (для продвинутых)

### Сменить цвета

Открой `css/styles.css`, найди:

```css
button { background: #2c3e66; }  /* основной цвет */
body.light { background: #f8fafc; }  /* фон светлой темы */
body.dark { background: #0f172a; }   /* фон тёмной темы */
```

Замени `#2c3e66` на свой цвет. Например, `#dc2626` для красного.

### Сменить название сайта

Открой `js/app.js`, найди `<h1>📘 База знаний <span class="subtitle">⚡ с умным поиском</span></h1>` — замени на своё.

### Добавить свои поля к документу

1. Открой `js/library.js` — функция `App.addDocument`
2. Добавь новое поле в объект `doc`
3. Открой `js/app.js` — функция `_renderHomeTab` — добавь отображение нового поля
4. Открой `firestore.rules` — добавь новое поле в `request.resource.data.keys().hasAll([...])`

## 💬 FAQ

**Q: Это бесплатно?**
A: Да, Firebase бесплатен в пределах Spark (free) плана. До 50 000 чтений/день и 20 000 записей/день. Для команды из 10-20 человек — за глаза.

**Q: Сколько юзеров можно добавить?**
A: На бесплатном плане — без ограничений. Но производительность упадёт после ~100 одновременно активных.

**Q: А если Firebase закроют?**
A: Все данные экспортируются: Firestore → Import/Export → Cloud Storage. Можно мигрировать.

**Q: Можно поменять Owner?**
A: Только пересоздать документ `config/owner` вручную в Firebase Console. Owner — постоянная роль.

**Q: А есть 2FA?**
A: Нет. По запросу заказчика не делали. Можно добавить позже.

**Q: Кто это сделал?**
A: Тёма попросил, AI написал код, Firebase хранит данные. 🤝
